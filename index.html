Database Best Practices

Transaction Handling

Recommended Pattern 
public function transferFunds(int $fromAccount, int $toAccount, float $amount): void
{
    DB::beginTransaction();
    try {
        // Transfer logic
        DB::commit();
    } catch (Exception $e) {
        DB::rollBack();
        Log::error('Transfer failed: ' . $e->getMessage());
        throw $e;
    }
}

Review Focus

  • Proper transaction handling
  • Error handling and rollback
  • Logging of failures

Security Best Practices

Input Validation & SQL Injection Prevention

Secure Code 
public function updateUser(Request $request): Response
{
    $data = $request->validate([
        'email' => 'required|email',
        'name' => 'required|string|max:255'
    ]);

    return User::findOrFail($request->validated('user_id'))
        ->update($data);

Use proper validation and prepared statements to prevent SQL injection.

Vulnerable Code 
$query = "UPDATE users SET email = '" . $_POST['email'] . "'
         WHERE id = " . $_GET['id'];
$result = mysqli_query($conn, $query);

Never use direct string concatenation with user input in queries.

Security Checklist

  • Input validation for all user inputs
  • Use prepared statements or query builders
  • Implement CSRF protection
  • Proper authentication checks
  • XSS prevention measures

Architecture & Design Patterns

SOLID Principles Implementation

Good Architecture 
interface PaymentGateway {
    public function processPayment(Payment $payment): bool;
}

class StripeGateway implements PaymentGateway {
    public function processPayment(Payment $payment): bool {
        // Stripe-specific implementation
    }
}

class PaymentService {
    private PaymentGateway $gateway;
    
    public function __construct(PaymentGateway $gateway) {
        $this->gateway = $gateway;
    }
}

Follow dependency injection and interface segregation principles.

Architectural Considerations

  • Service layer implementation
  • Dependency injection usage
  • Interface segregation
  • Single responsibility principle
  • Code modularity

Testing Guidelines

Unit Testing

Test Example 
public function test_user_creation(): void
{
    $userData = [
        'name' => 'John Doe',
        'email' => 'john@example.com'
    ];

    $user = User::create($userData);

    $this->assertDatabaseHas('users', $userData);
    $this->assertEquals($userData['name'], $user->name);
    $this->assertEquals($userData['email'], $user->email);
}

Write comprehensive tests covering main functionality and edge cases.

Testing Requirements

  • Unit tests for new features
  • Integration tests for complex flows
  • Test edge cases and error scenarios
  • Maintain test coverage standards
  • Mock external dependencies
